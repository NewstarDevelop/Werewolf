# 自动化体检-定位与修复

生成时间：2025-12-27 12:25:25

## 目标（可验证）
- 前端：`npm run lint`、`npm run build` 均通过
- 后端：`python -m compileall "app"` 通过（语法/导入级错误为 0）
- Docker：`docker compose config` 通过；容器启动正常；`verify_docker.ps1` 关键项一致
- API：`GET /health` 正常；基础“开始游戏/推进/行动提交”不报错；若可自动跑完整局则断言结束态 `players.role` 均非空

## 约束与原则
- KISS/YAGNI：仅实现体检与回归所需的最小改动
- DRY：复用现有脚本（`verify_docker.ps1`、`verify_api.ps1`），必要时补充最小化回归脚本
- SOLID：修复以“定位根因”为优先，不做无关重构

## 执行步骤
1. 读取基线配置：`docker-compose.yml`、`backend/Dockerfile`、`frontend/Dockerfile`、`frontend/vite.config.ts`、`backend/app/main.py`
2. 前端质量门禁：在 `frontend/` 下执行 `npm run lint`、`npm run build`，失败则最小化修复
3. 后端静态体检：在 `backend/` 下执行 `python -m compileall "app"`，失败则修复语法/导入级问题
4. Docker 体检：`docker compose config` → `docker compose up -d --build` → `.\verify_docker.ps1`
5. API 冒烟：`/health`、`/api/game/start`、`/api/game/{id}/step`
6. 回归验证：若需要稳定验证“结束态 role”，补充最小 PowerShell 回归脚本并纳入体检流程
