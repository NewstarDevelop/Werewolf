# 前端-调试面板与发言按钮重叠修复

生成时间：2025-12-27 13:38:04

## 背景与问题
- 当前页面存在“调试面板入口（系统日志）”与“发言发送按钮”在部分视口下发生遮挡/重叠的风险（入口按钮使用 `position: fixed` 位于底部右侧）。
- 同时需要对前端页面做一次完整质量检查，找出其它明显问题。

## 目标（可验证）
- 调试面板入口不再覆盖底部发言发送按钮/底部操作栏（移动端/窄屏更易触发）。
- `frontend/`：`npm run lint` 无 error；`npm run build` 通过。

## 方案概述
- 将调试面板入口从底部 `fixed` 浮动按钮迁移到顶部状态栏（`GameStatusBar`）右侧区域，避免与底部交互区抢占空间。
- 保持 `LogPanel` 行为不变（仍为右侧 `Sheet`）。

## 变更点（原子操作）
1. `frontend/src/components/game/GameStatusBar.tsx`
   - 增加可选回调 `onOpenLogs?: () => void` 并渲染“系统日志”按钮（仅当回调存在时展示）。
2. `frontend/src/pages/Index.tsx`
   - 移除底部右侧 `fixed` 的日志按钮
   - 将 `setLogPanelOpen(true)` 通过 `GameStatusBar` 的 `onOpenLogs` 触发
3. 质量门禁
   - 执行 `npm run lint`、`npm run build`
